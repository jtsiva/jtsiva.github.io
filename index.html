About me

Hi! With a background in computer science and a dedication to making learning more accessible, I strive to make a difference in how learners engage with STEM topics.

Resume

In developing learning experiences, I lean toward a mastery learning approach that is supported and informed by Universal Design for Learning. Helping develop learning opportunities that open the door to learning through varied forms of representation and expression is my primary goal. My Computer Science background has steered me toward applying my LD skills within technological domains for which I can more directly support Subject Matter Experts, but I also enjoy harnessing my background to prototype new tools for the learning and design processs.

AI-powered Course Development

/

As part of an experimental GenAI-centered project, I developed a workflow to ensure that screencast scripts were based on quality code. Moreover, the generated screencast script is intended to include code snippets and sufficient detail to be able act as as alternative representation of the screencast for those who need it. The process was this:

  • create very basic outline of a screencast
  • Use a GenAI agent to help create a python notebook
  • LD gets notebook operational
  • SME reviews and updates notebook
  • Use notebook to generate script
  • SME reviews and updates script!

Recording Guidance for Screencasts

Created concrete recording guidance for instructors to help them create more accessible screencast videos. I created my own demonstration videos to clearly illustrate best practices including zooming in, changing cursor color and size, and using more descriptive language to describe on-screen actions.

Expanded on a set of guidance slides by creating a video that illustrated the different elements required for this type of learning experience. This served the two-fold purpose of establishing a common vocabulary as well as providing concrete exampels of what we were looking for.

Narrated Slide Widget

I have been playing around with the idea of a slideshow that can accompany narration and change slides on cue. This type of learning item is fairly common, but the text will not be interactible which is a major drawback. Here is a prototype of the idea using Remark to create interactible slides using markdown. As the audio plays, the current slide will adjust. They will also adjust if the audio player slider is moved. Note that this a work in progress.
